[L0002] Freeze a ratified oracle with a manifest hash + Layer-0 test, not honor-system | candidate | added: 2026-07-08 | tags: eval-lessons, gate-discipline, patent-domain | lesson: When a golden/oracle set is ratified, make "the oracle is sacred" mechanical: sha256 the canonical form of ONLY the ratified item ids (`layer_e.ratified_manifest_sha256`), stamp it in a `ratified` block, and add a Layer-0 test that recomputes + compares. Hashing only the frozen ids (not the whole list) is what keeps it APPEND-ONLY — new items may be added freely; frozen ones can't change without a re-stamp. The test must read only the JSON (no corpus store) so it runs in CI (conftest.py auto-marks everything under tests/ as layer0), not just locally. Prove it bites before trusting it: tamper→fail, append→pass, delete→caught. | evidence: 2026-07-08 D23 ratified golden_patent.json (25 items); all three properties proven; CI run 28967191374 green. | falsifier: a future need to freeze per-field rather than whole-item (e.g. cosmetic rationale edits shouldn't break the gate) — then hash only the label-bearing fields. | supersedes: —
[L0003] Google Patents serves each drawing sheet as thumbnail AND full-res under the same name — pick by size | candidate | added: 2026-07-08 | tags: patent-domain, harness-lifecycle | lesson: When scraping patent figures (or any asset that comes in multiple resolutions under one filename), don't take the first URL match — you silently get the 82×120 thumbnail.
